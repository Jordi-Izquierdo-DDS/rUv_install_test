## Bugs found

### 1. VerdictAnalyzer null error (Fix 18 bug)

```
Failed to convert JavaScript value `Null` into rust type `String` on JsTrajectoryStep.error
```

When step has `success: true`, JS sends `error: null`. Rust napi_simple.rs defines `error: Option<String>` but NAPI-RS fails to convert JS null to Rust None for this field. Impact: VerdictAnalysis returns null for successful trajectories. Only failure trajectories with explicit error strings work.

Fix: In napi_simple.rs, handle the null case explicitly or change the field type.

### 2. Session 1 routing regression (6/10 vs prior 7/10)

Intra-session pattern accumulation causes instability. As patterns are stored during routing, subsequent queries in the same session get boosted/penalized by earlier patterns. This can WORSEN routing within a single session before the k-means clustering stabilizes.

### 3. Consistency: first query differs

Run 1 of "fix authentication vulnerability" returns backend-developer (cosine base). Runs 2-5 return security-architect (after pattern stored by run 1 feeds back). This is learning-as-designed but confusing.

### 4. HNSW route-index not persisting

`routesPersisted: undefined` in session_end. The HNSW service (Fix 16) is now redundant with SonaEngine patterns carrying modelRoute (Fix 17). Should be removed or fixed.

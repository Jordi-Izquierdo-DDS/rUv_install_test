## Root cause of 3 routing failures

1. **auth/JWT → backend (not security):** `backend-developer` pattern includes "authentication" → cosine sim 99.6% vs security-architect 99.5%. Gap = 0.1%. Boost +0.05 can't flip.
2. **unit tests → coder (not tester):** "write" + "module" → "implement build create code" (coder) wins over "test unit integration" (tester). "write" = coder semantically.
3. **SQL injection → backend (not security):** "database" in backend pattern attracts "SQL". Same as #1 — security-architect is close second but loses.

## Root cause of IMPROVEMENT = 0

The learning loop is **complete but circular:**
1. Route assigns `backend-developer` for auth query
2. Trajectory is recorded with `backend-developer` as the agent
3. Pattern crystallizes with `backend-developer` metadata
4. Next session: findPatterns returns patterns tagged `backend-developer`
5. Boost +0.05 goes to `backend-developer` (reinforcing the wrong answer)

**Learning reinforces existing routing decisions, correct or not.** Without negative feedback ("that was wrong, should be X"), patterns can only strengthen the status quo.

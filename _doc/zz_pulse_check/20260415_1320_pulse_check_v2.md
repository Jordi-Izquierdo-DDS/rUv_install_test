## 4. Three options to advance

### Option A: Accept & log

- **Action**: document 5-layer §2 evidence in §12 DQ log; no code changes
- **DQ resolved**: none
- **Effort**: 0
- **Valid when**: the DQ issues don't empirically block UX (we CAN retrieve patterns by similarity, just without categorical distinction)

### Option B: NAPI rebuild of ruvllm (Path 3, same pattern as sona rebuild 2026-04-15)

- **Action**: add `#[napi]` annotations to `VerdictAnalyzer`, `ReasoningBank`, `PatternCategory`, `Verdict`, `Trajectory` in `examples/ruvLLM/src/napi.rs`; `cargo build`; vendor `.node`
- **DQ resolved**: 03 (direct), 06 (probably — task-boundary needs diverse signal which categorization enables), partial 01 (patterns would have variance in avg_quality across categories)
- **Effort**: `examples/ruvLLM` crate is much larger than sona (has inference/attention/kernels/GGUF/SIMD deps); expected rebuild time > sona's 16s; output .node several MB
- **Valid when**: DQ-03 is empirical blocker

### Option C: Adopt `ruvector.IntelligenceEngine` orchestrator

- **Action**: refactor daemon to use `IntelligenceEngine` as primary service (wraps same `@ruvector/sona` we use, adds richer API)
- **DQ resolved**: none directly (same sona internal → same categorization gap)
- **Gains**:
  - **Phase 3 APPLY routing activated** (currently deferred per ADR-005): `IntelligenceEngine.route(task, file)` + `routeWithWorkers()` work out of the box
  - **Phase 10 CONSOLIDATE partial**: `recordEpisode`/`queueEpisode`/`flushEpisodeBatch`/`remember`/`recall` — episodic memory primitives
  - **Specialized tracking**: `recordCoEdit`, `recordErrorFix`, `getSuggestedFixes`, `getLikelyNextFiles`, `getAlternates`
  - **Cleaner API**: `endTrajectory(success, quality?)` takes boolean directly; upstream owns 0.9/0.3 fallback (our hook-handler doesn't need the `0.5` passthrough as explicit invention)
  - Uses same NAPI sona we already have; same xenova monkey-patch works (confirmed: intelligence-engine.js line 21 imports onnx-embedder)
- **Effort**: refactor daemon ~60 LOC, preserve ADR-007 services pattern, new service `intelligence` replacing or composing with `sona` service
- **Valid always** (architectural improvement, independent of DQ-03)

## 5. Recommendation

**Sequential: C first, B only if dogfood forces it.**

1. **Adopt Option C immediately.** Value proposition: free Phase 3 APPLY + episodic memory + cleaner API, without any rebuild. ADR-005 Phase 3 APPLY deferral flips from ☐ to ☑.
2. **Update §12 DQ log** with full 5-layer §2 evidence. DQ-03 stays ☐ but now definitively classified as "upstream binding gap, not ruflo limitation".
3. **Defer Option B** until empirical forcing function: either dogfood across heterogeneous projects reveals that the lack of categorization causes retrieval UX issues, OR the operator decides to invest upstream-side capability.
4. **Do NOT implement Option B speculatively.** ADR-005 §7 explicitly requires empirical forcing function (operator CRITICAL flag or dogfood evidence) before vendor rebuilds. Academic curiosity ≠ forcing function.

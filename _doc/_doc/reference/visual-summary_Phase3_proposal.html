Historical narrative (kept for forensic context — the reasoning that led to the 2026-04-15 rebuild decision):

A user asked: "if capture works but patterns don't emerge, is ruflo only partially capturing?" The answer — after reading crates/sona/src/loops/{instant,background}.rs and running ablations against the v4 daemon — is that ruflo's capture path is complete. What was gated was pattern distillation, by a two-layer upstream gate inside @ruvector/sona@0.1.5.

Layer 1 — min_trajectories gate

In background.rs::run_cycle, force_learn returns "skipped: insufficient trajectories" when trajectories.len() < min_trajectories. Upstream v2.1.2 lowered this to 10 (source: crates/sona/src/loops/background.rs:30, comment: "Was 100; lowered so patterns crystallize from fewer trajectories"). Pinned @ruvector/sona@0.1.5 still uses 100 (pre-fix). Ablation: n=15 skipped, n=100 completed — gate confirmed at 100.

Layer 2 — k-means cluster quality

Even at n=100 (gate cleared, status: "completed"), bank.extract_patterns() returned 0 clusters. Second-layer gate: k-means in reasoning_bank requires semantic diversity + per-cluster quality threshold. Synthetic smoke prompts ("refactor variant N") are too similar — embeddings cluster too tightly. Requires varied real-world prompts to crystallize.

Both gates clear given enough trajectories AND semantic diversity.

What IS working: capture (120/120 buffered, 0 dropped), step accumulation, MicroLoRA gradient accumulation (fires on every on_trajectory), pattern distillation at scale, C4 SQLiteBackend.store (persisted, verified across restart).

C4 STORE integrity verified: initial smoke-04 read showed 100 entries out of 120, but follow-up re-query with limit: 500 returned 120/120. Root cause was a self-imposed memory_query IPC clamp capped at 100 in the daemon handler; lifted to 1000 post-ablation. No upstream maxEntries cap, no silent store failure.

EWC++ re-assessment (2026-04-14): previously marked OQ-3 as "EwcPlusPlus.consolidate not exposed → catastrophic-forgetting protection missing". Source re-read (crates/sona/src/ewc.rs + loops/background.rs::run_cycle) shows the truth is much narrower: update_fisher, detect_task_boundary, start_new_task (task-boundary consolidation), and apply_constraints all run automatically inside force_learn→run_cycle. Incremental EWC++ IS protecting against forgetting. The only missing piece is the cross-task compaction call consolidate_all_tasks() — a ≈5-line NAPI addition, not a major gap.

OQ-2 outcome (2026-04-15): path (b) taken — built from ruvector_GIT_v2.1.2_20260409 via scripts/rebuild-sona.sh. First pattern now crystallizes from ≥3 trajectories. min_trajectories gate is history.

OQ-3 outcome (2026-04-15): path (b) taken — added 2 #[napi] annotations to napi_simple.rs in the same rebuild (consolidate_tasks + prune_patterns, ≈30 LOC wrapper).

2. Graceful Degradation

The system never crashes. Each component failure degrades to a simpler mode that still works.

LevelWhat failsWhat still worksRoutingLearningImpact
Full stack Nothing SR + sona patterns + rbank quality + VerdictAnalyzer 8/10 3 loops + VA Best: routing improves over sessions
-rbank @ruvector/ruvllm-native missing SR + sona patterns (no VerdictAnalyzer) 7/10 3 loops (no VA quality) Routing still improves, no root cause analysis
-sona @ruvector/sona NAPI missing SemanticRouter alone (no learned patterns) 8/10 None Good routing, no learning — static accuracy
-SR @ruvector/router missing Cosine against AGENT_PATTERNS (11 agents) 7/10 3 loops if sona present Decent routing, known overlap issues (auth/backend)
-ONNX @xenova/transformers missing Hash fallback (13% embedding density) ~3/10 Poisoned Barely functional — logged, visible in [INTELLIGENCE]
-daemon Daemon won't start / crashes Hooks run, preBashSafety works, no IPC N/A None Pass-through: Claude Code works normally, no intelligence

Key principle: each layer's failure is caught by try/catch + logged. No exception propagates to crash the daemon or block Claude Code hooks. The handler has a 5s global timeout — if the daemon is unresponsive, hooks exit cleanly.

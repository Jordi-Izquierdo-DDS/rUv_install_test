§1 · RuVector is a 7-layer stack (from foxref part01)

Layer 7: AGENT FRAMEWORK (rvAgent) McpServer, ToolRegistry, Middleware pipeline, SonaMiddleware — Rust-only, no JS binding Layer 6: COGNITIVE CONTAINERS (RVF) .rvf files, MicroVm::launch — DEFERRED (feedback_no_rvf.md) Layer 5: SELF-LEARNING (SONA) SonaEngine (3 loops) + TrajectoryBuilder + ReasoningBank + EwcPlusPlus — WE USE (sona NAPI) Layer 4: INTELLIGENCE 58+ attention mechs, GNN (GAT/GraphSAGE/GraphMAE), MinCUT (165 symbols x 49 modules) Prime Radiant (CoherenceMemory 80 downstream) + Consciousness Phi — accessible via NeuralSubstrate Layer 3: DISTRIBUTED (Raft) Consensus, replication — N/A (single-process) Layer 2: POSTGRES EXTENSION (230+ SQL funcs) — N/A (single-node) Layer 1: VECTOR CORE HnswIndex, VectorDB, SIMD distance — transitive via IntelligenceEngine + sona

We correctly use L5 (sona). L4 (NeuralSubstrate bundle) is the missing high-value layer accessible from JS without rebuild. L7 SonaMiddleware is Rust-only; we replicate its effect via direct hook-handler calls.

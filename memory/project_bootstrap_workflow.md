This project (`rufloV3_bootstrap_v3_CGC`) is a **portable bootstrap kit** — an installer that sets up the v3 learning system in OTHER projects.

**Where changes go:**
- `scripts/bootstrap.sh` — the installer script
- `scripts/patches/` — upstream bug fixes
- `scripts/templates/helpers/` — the v3 files (hook-handler.cjs, sona-hook-handler.mjs, ruvector-runtime-daemon.mjs, ruvector-ipc-client.mjs, hook-bridge.cjs)
- `scripts/patches/verify-v3.sh` — verification
- `README.md`, `__CGC_GAP_Analysis_v2_/` — documentation

**Where changes do NOT go:**
- `.claude/helpers/` — this is project-local, generated by the installer. If it needs updating, re-run the bootstrap.
- `node_modules/`, `.swarm/`, `.ruvector/` — runtime artifacts, not part of the kit.

**Workflow:**
1. Make changes in `scripts/` (the portable part)
2. Test by installing into a fresh project (e.g., v202): `cp bootstrap.sh scripts/ /target/ && cd /target && ./bootstrap.sh`
3. Once verified, sync documentation (README.md, gap analysis)
4. The project-local `.claude/helpers/` is only updated by re-running the bootstrap — never edit it directly as the source of truth.

**Why:** The deliverable is the installer, not this project's local installation.
